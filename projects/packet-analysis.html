Project Overview

This project focused on capturing and analyzing network traffic to understand how packets flow through a network and how to diagnose common connectivity issues. Using Wireshark as the primary analysis tool, I captured over 200 packets across various protocols including ICMP, ARP, and TCP.

The goal was to move beyond theoretical knowledge and actually see what happens at the packet level when you ping a device, when ARP resolves MAC addresses, or when a connection fails. This hands-on approach gave me practical troubleshooting skills that I can apply to real-world network issues.

Problem Solved

‚ö†Ô∏è

The Problem

One device in my lab network couldn't ping the gateway. The error said "Request timed out" but all cables were connected and IP configuration looked correct.

‚Üí
‚úì

The Solution

Wireshark showed ARP requests going out but no replies coming back. The gateway interface was in shutdown state. Once I enabled it, ARP replies came through and connectivity was restored.

What I Did

1

Set Up the Lab Environment

Created a simple network topology in GNS3 with 2 PCs, 1 switch, and 1 router to simulate a basic LAN setup.

2

Started Packet Capture

Opened Wireshark on the interface and began capturing all traffic without filters to see everything happening on the wire.

3

Generated Network Traffic

Used ping commands, accessed different devices, and intentionally created scenarios to generate various types of traffic.

4

Applied Display Filters

Used Wireshark filters like "icmp" and "arp" to isolate specific traffic types and focus on what I wanted to analyze.

5

Analyzed Packet Details

Clicked on individual packets to examine source, destination, protocol information, and payload data at each OSI layer.

6

Diagnosed the Issue

Identified that ARP requests weren't getting replies, which led me to discover the shutdown interface on the router.

Commands & Filters Used

Wireshark Display Filters
icmp                     # Show only ICMP packets
arp                      # Show only ARP packets
ip.addr == 192.168.1.1   # Filter by IP address
tcp.port == 80           # Filter by TCP port
eth.addr == aa:bb:cc     # Filter by MAC address
ip.src == 192.168.1.10   # Filter by source IP
Network Commands
ping 192.168.1.1         # Test connectivity
arp -a                   # View ARP table
tracert 192.168.1.1      # Trace packet path
ipconfig /all            # Show IP configuration
netstat -an              # Show active connections
