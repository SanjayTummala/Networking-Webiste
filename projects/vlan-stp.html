Project Overview

This project focused on two critical Layer 2 technologies: VLANs for network segmentation and Spanning Tree Protocol (STP) for loop prevention. I created three VLANs on switches to separate HR, IT, and Guest traffic - even though all devices connect to the same physical switches, they're logically isolated and can't see each other.

I also connected switches in a redundant loop topology to test STP. Without STP, broadcast storms would kill the network within seconds. STP automatically detects the loop and blocks one port to break it, while keeping the path available as a backup if the primary link fails.

Problem 1: No Segmentation

Solution: VLANs

Created separate broadcast domains. HR can't even see IT devices in their network neighborhood. Complete logical separation.

Problem 2: Switching Loops

Connected switches for redundancy but broadcasts started looping infinitely. Network crashed within seconds due to broadcast storm.

Solution: STP

STP elected a root bridge and blocked one port to break the loop. Redundancy exists but the loop is prevented.

VLAN Configuration

I created three VLANs to segment the network by department. Each VLAN has its own IP subnet and dedicated switch ports:

STP Port States

When STP is enabled, switch ports go through several states before forwarding traffic. This prevents temporary loops during topology changes:

Blocking
20 seconds
Listening
15 seconds
Learning
15 seconds
Forwarding
Active

Total convergence time: ~30-50 seconds (can be reduced with Rapid STP)

Key Learnings

üîÄ

Access vs Trunk

Access ports carry one VLAN untagged. Trunk ports carry multiple VLANs with 802.1Q tags.

üëë

Root Bridge

STP elects one switch as root based on lowest Bridge ID. All path costs calculated from root.

üö´

Blocked Ports

STP blocks redundant ports to prevent loops. These activate automatically if primary fails.

üîó

Inter-VLAN Routing

VLANs can't communicate without a Layer 3 device (router or L3 switch). Isolation is the point.
