Project Overview

This project focused on manually configuring routes on routers to connect multiple networks and ensure packets reach their destinations. I connected 3 routers in GNS3, each with its own network segment, and configured static routes so all networks could communicate.

Without routing configured, Network A couldn't talk to Network C because routers didn't know where to send packets. I learned to add static routes that tell each router exactly where to forward traffic for specific destination networks, and default routes for traffic going to unknown destinations.

The Problem

PC1 in Network A (192.168.1.0) couldn't ping PC3 in Network C (192.168.3.0). Router R1 had no idea how to reach Network C - the packets were being dropped.

‚Üí
‚úì

The Solution

Added static routes on each router pointing to remote networks via the next-hop router. Now all three networks can communicate with each other.

Project Objectives

  • ‚úì Build a multi-router topology connecting three separate networks
  • ‚úì Configure IP addresses on all router interfaces
  • ‚úì Add static routes for each remote network destination
  • ‚úì Configure default routes for edge routers
  • ‚úì Verify end-to-end connectivity with ping and traceroute
  • ‚úì Understand routing table structure and route selection

What I Did

1

Built the Topology

Created 3 routers connected in a line, with each router having one LAN network and a PC attached.

2

Assigned IP Addresses

Configured IP addresses on all interfaces. Router-to-router links used 10.0.x.0/30 subnets, LANs used 192.168.x.0/24.

3

Added Static Routes

On R1, added routes for 192.168.2.0/24 and 192.168.3.0/24 via R2. Repeated for all routers in both directions.

4

Configured Default Route

Added 0.0.0.0/0 default route on edge routers for traffic going to unknown destinations (simulated internet).

5

Tested Connectivity

Pinged from PC1 to PC3 and verified success. Used traceroute to confirm packets took the correct path.

6

Verified Routing Tables

Used "show ip route" to examine the routing table and confirm all routes were properly installed.

Configuration Commands

Static Route Syntax
! Syntax: ip route [dest] [mask] [next-hop]

! R1 Configuration
R1(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2
R1(config)# ip route 192.168.3.0 255.255.255.0 10.0.0.2

! R3 Configuration  
R3(config)# ip route 192.168.1.0 255.255.255.0 10.0.1.1
R3(config)# ip route 192.168.2.0 255.255.255.0 10.0.1.1
Default Route
! Send all unknown traffic to next-hop
R1(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.2

! This catches all traffic not matching
! specific routes in the routing table
Interface Configuration
R1(config)# interface g0/0
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown

R1(config)# interface g0/1
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# no shutdown
Verification Commands
R1# show ip route
R1# show ip route static
R1# ping 192.168.3.10
R1# traceroute 192.168.3.10

Key Learnings

üó∫Ô∏è

Routing Tables

Routers use tables to decide where to send packets. No matching entry means packet gets dropped.

‚ÜîÔ∏è

Two-Way Routes

Ping needs a reply, so routes must exist in both directions or communication fails.

üéØ

Next-Hop Logic

Router doesn't know the full path. It just knows "send to this neighbor" and trusts the next router.

‚öôÔ∏è

Static vs Dynamic

Static routes don't adapt to changes. Good for small networks, impractical for large ones.
