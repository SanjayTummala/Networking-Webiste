Project Overview

This project involved configuring OSPF (Open Shortest Path First) across a network of 4 routers. Unlike static routing where I manually added each route, OSPF routers automatically discover each other, share network information, and calculate the best paths to all destinations.

The most impressive part was testing failover - when I disconnected one link, OSPF detected the failure through missed hello packets and automatically recalculated routes to use an alternate path. The network recovered in about 30 seconds without any manual intervention.

üéØ Real-Time Application

This project simulates how enterprise networks use dynamic routing protocols to maintain connectivity even when individual links or devices fail. It's the foundation for building resilient, self-healing networks.

OSPF Neighbor States

OSPF routers go through several states before they can exchange routes and form a full neighbor relationship:

DOWN
No hellos received
INIT
Hello received
2-WAY
Bidirectional
EXSTART
Master/Slave
EXCHANGE
DBD exchange
LOADING
LSA request
FULL
Synchronized

Configuration Commands

Enable OSPF
Router(config)# router ospf 1
Router(config-router)# router-id 1.1.1.1
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0
Router(config-router)# network 10.0.0.0 0.0.0.255 area 0

! Wildcard mask is inverse of subnet mask
! 0.0.0.255 matches any IP in /24 network
OSPF Cost Adjustment
Router(config)# interface g0/0
Router(config-if)# ip ospf cost 10

! Lower cost = preferred path
! Default cost = 100Mbps / interface bandwidth
! GigabitEthernet = cost 1
! FastEthernet = cost 10
Verification Commands
Router# show ip ospf neighbor
Router# show ip ospf interface brief
Router# show ip ospf database
Router# show ip route ospf
Test Failover
Router(config)# interface g0/1
Router(config-if)# shutdown

! Watch OSPF detect failure (dead timer = 40s)
! See routes recalculate automatically
! Verify traffic uses alternate path

Key Learnings

üëã

Hello Protocol

OSPF sends hello packets every 10 seconds. If 4 hellos are missed (40s), neighbor is declared dead.

üí∞

Cost Metric

OSPF picks paths based on cumulative cost. Faster links have lower cost - Gigabit preferred over FastEthernet.

üóÇÔ∏è

Link State Database

Every router maintains a complete map of the network topology. All routers have identical LSDBs.

‚ö°

Fast Convergence

When topology changes, OSPF recalculates in seconds. Much faster than waiting for manual fixes.
